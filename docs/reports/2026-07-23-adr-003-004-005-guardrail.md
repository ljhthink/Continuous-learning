### R1. M-1 复核（必须修正项）— 未通过，阻断

**核心结论：ADR-005:39 的修正未真正修复 bug，M-1 仅修正了一半。**

主 Agent 对 M-1 的修正分两处：

| 位置 | 修正方式 | 实际运行结果 | 判定 |
| --- | --- | --- | --- |
| [ADR-005:39](../decisions/ADR-005-public-vs-private-repository.md#L39)（D2 表格） | 单引号包裹，但**保留反斜杠转义**（见下方命令 A） | **匹配 0 行**（假阴性） | 未修复 |
| [ADR-005:107](../decisions/ADR-005-public-vs-private-repository.md#L107)（验证章节） | 单引号包裹，**移除反斜杠转义**（见下方命令 B） | 匹配 26 行（全部为讨论性提及，无真实凭证） | 已修复 |

**取证证据（本审计实际运行）**：

- 命令 A（ADR-005:39 当前写法，保留 `\|`）：

  ```text
  rg -i 'api[_-]?key\|secret\|token\|password' --type-add 'cfg:*.{ts,js,json,toml,yml,yaml,md}' -t cfg
  ```

  输出：**0 行匹配**。ripgrep 正则引擎将 `\|` 解释为**字面量管道符 `|`**（Rust regex 语义：`\|` = 转义的 `|`），故该正则实际搜索的是含字面 `|` 的完整字符串 `api[_-]?key|secret|token|password`。代码库无此字面串，故 0 匹配——这是**假阴性**：即使存在真实密钥也找不到。

- 命令 B（ADR-005:107 当前写法，移除 `\`）：

  ```text
  rg -i 'api[_-]?key|secret|token|password' --type-add 'cfg:*.{ts,js,json,toml,yml,yaml,md}' -t cfg
  ```
